GPIO Numbering

All GPIO numbers in this guide use BCM numbering, not the physical board pin number.

To see both, run: pinout on your Raspberry Pi terminal.

I2C must be enabled via sudo raspi-config.

How the Scan Works

Rows are driven HIGH one at a time. Columns are read as inputs with internal pull-down resistors. A key press shorts its row wire to its column wire — the column reads HIGH while that row is active.

Example: User presses key "5" (Row 2, Col 2) 1. Drive GPIO 27 (R2) → HIGH 2. Read GPIO 9 (C1) → LOW (no connection) 3. Read GPIO 11 (C2) → HIGH ← key pressed! KEY_MAP[1][1] = '5' 4. Drive GPIO 27 (R2) → LOW 5. Continue to next row…

Component

Model: UCB Mini v4 coin acceptor

Input Voltage: 12V DC

Signal Output: 5V pulse (one pulse per peso)

Accepted: ₱1 (1 pulse), ₱5 (5 pulses), ₱10 (10 pulses)

Signal Timing

Debounce: 30 ms (ignores bounces shorter than this)

Pulse gap: ~50–80 ms between pulses (UCB Mini v4)

Finalize timeout: 3.0 s of silence → total is calculated

Default polarity: FALLING edge (optocoupler inverts signal)

⚠️ CRITICAL: The UCB Mini v4 outputs a 5V signal. The Raspberry Pi GPIO is 3.3V tolerant only. You MUST use a voltage divider before connecting to GPIO — direct connection will permanently damage the GPIO pin.
Voltage Divider Circuit (5V → 3.3V)
UCB Mini v4 Raspberry Pi SIG (5V) ──────── 1kΩ ──────┬──────── GPIO 4 (Pin 7) │ 2kΩ Vout = 5 × 2/(1+2) = 3.33V ✓ │ GND
Use 1kΩ and 2kΩ resistors. Any equivalent ratio (e.g. 2kΩ + 4kΩ) works as long as the output stays ≤ 3.3V.

Signal Polarity

FALLING (default)

Signal rests HIGH, goes LOW on each coin pulse.

Typical for boards with an optocoupler that inverts the signal.

Set in .env: COIN_EDGE=FALLING

RISING

Signal rests LOW, goes HIGH on each coin pulse.

Typical for direct/buffer connections without signal inversion.

Set in .env: COIN_EDGE=RISING

Run sudo venv/bin/python test_coin_signal.py to auto-detect which polarity your board uses before configuring.

Purpose

The relay cuts power to the coin acceptor when no PC is selected. This prevents coins from registering when no session is pending, and stops the coin acceptor drawing 12V power unnecessarily.

Behavior

GPIO 6 HIGH → Relay ON → Coin acceptor powered → accepts coins

GPIO 6 LOW  → Relay OFF → Coin acceptor unpowered → rejects coins

Relay activates only in AWAITING_COINS state, deactivates on all other transitions.

Configuration (.env)

I2C Address

If the display doesn't work, check its actual address with sudo i2cdetect -y 1 and update:

LCD_I2C_ADDRESS=0x3F

Contrast Adjustment

If characters are visible but dim or completely blank, adjust the small blue potentiometer on the I2C backpack board with a small screwdriver.

Keypad Issues
⌨️

Keypad Troubleshooting

▶
SymptomLikely CauseFix
Entire row dead (e.g. 1,2,3 all fail) Row wire disconnected or wrong GPIO Re-check the row GPIO connection for that row. Run test_keypad.py.
Entire column dead (e.g. 3,6,9,# all fail) Col wire disconnected or wrong GPIO Re-check the column GPIO connection. Update KEYPAD_COLS in .env if needed.
Wrong key registered when a key is pressed Row or column wires in wrong order Run test_keypad.py, compare actual vs expected output, swap GPIO pins in config.
Two separate columns/rows both fail (e.g. 3,6,9 and *,0 fail but # works) Two wires swapped (e.g. R4 ↔ C3 connector pins) Swap KEYPAD_ROWS[3] and KEYPAD_COLS[2] in config.py or .env.
No keys detected at all RPi.GPIO not installed or I2C/SPI conflict Run sudo pip install RPi.GPIO. Check server logs for GPIO errors.
Key registers multiple times per press Scan interval too fast or no debounce Increase KEYPAD_SCAN_INTERVAL in .env (default 0.05s).
Coin Slot Issues
🪙

Coin Slot Troubleshooting

▶
SymptomLikely CauseFix
No coin detected at all Wrong signal polarity Run test_coin_signal.py, set COIN_EDGE=RISING or FALLING accordingly in .env.
Coin detected but wrong count (e.g. ₱5 shows as ₱1) Debounce too aggressive — swallowing pulses Lower COIN_DEBOUNCE_MS (try 20ms) in .env.
One coin counted multiple times Debounce too loose — counting noise Raise COIN_DEBOUNCE_MS (try 40ms) in .env.
GPIO overvoltage warning or pin damaged 5V signal connected directly to GPIO Add voltage divider (1kΩ + 2kΩ). GPIO max is 3.3V.
Coin accepted when no PC is selected Relay not cutting power Check relay wiring and GPIO 6 signal with a multimeter.
Coin acceptor never powers on Relay wired to NC instead of NO Move the 12V wire from NC contact to NO contact on the relay.
