🔐 Authentication No Auth Required
POST /api/auth/token Get admin JWT token ▶

Authenticates admin credentials and returns a JWT Bearer token. Include this token as Authorization: Bearer <token> on all admin API requests. Token expires in 8 hours.

Request Body (application/x-www-form-urlencoded)
FieldTypeRequiredDescription
usernamestringrequiredAdmin username
passwordstringrequiredAdmin password
Request
curl -X POST http://192.168.1.100:8000/api/auth/token \
  -d "username=admin&password=admin123"
Response 200 OK
{"access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "token_type": "bearer"}
Response 401 Unauthorized
{"detail": "Incorrect username or password"}
🖥 PC Endpoints No Auth
POST /api/pc/register Register or reconnect a PC ▶

Called by the VB.NET client on startup. Registers the PC on first launch. On subsequent calls it updates the IP address and MAC. Returns whether this was a fresh registration.

Query Parameters
ParamTypeRequiredDescription
pc_numberintrequiredUnique PC number (e.g. 1, 2)
mac_addressstringrequiredMAC address of the client machine
Request
curl -X POST "http://192.168.1.100:8000/api/pc/register?pc_number=1&mac_address=AA:BB:CC:DD:EE:FF"
Response 200 OK — new PC
{"pc_id": 1, "pc_number": 1, "name": "PC-01", "registered": true}
Response 200 OK — already registered (IP/MAC updated)
{"pc_id": 1, "pc_number": 1, "name": "PC-01", "registered": false}
POST /api/pc/heartbeat/{pc_number} Client heartbeat — syncs session state ▶

Sent by the client every 10 seconds. Marks the PC as online and returns the authoritative session state. If time_added_minutes > 0 the client shows a toast notification. If is_locked: true and the local timer hits zero, the client locks the screen.

Path Parameters
ParamTypeDescription
pc_numberintThe PC's number
Request
curl -X POST http://192.168.1.100:8000/api/pc/heartbeat/1
Response 200 OK — Active session
{"is_locked": false, "remaining_minutes": 28, "remaining_seconds": 45, "session_token": "tok_abc123", "time_added_minutes": 0}
Response 200 OK — No session / locked
{"is_locked": true, "remaining_minutes": 0, "remaining_seconds": 0, "session_token": null, "time_added_minutes": 0}
Response 200 OK — Admin just added time
{"is_locked": false, "remaining_minutes": 60, "remaining_seconds": 0, "session_token": "tok_abc123", "time_added_minutes": 30}
Response 404 — PC not registered
{"detail": "PC not registered"}

⏱ Session Endpoints No Auth
POST /api/session/add-time Add time by inserting coins (pesos → minutes) ▶

Called by the hardware controller when coins are inserted. Converts pesos to minutes using the configured coin rates, creates or extends the session, and unlocks the PC.

Request Body (application/json)
FieldTypeRequiredDescription
pc_numberintrequiredTarget PC number
pesosintrequiredTotal coins inserted (e.g. 5, 10)
Request
curl -X POST http://192.168.1.100:8000/api/session/add-time \
  -H "Content-Type: application/json" \
  -d '{"pc_number": 1, "pesos": 5}'
Response 200 OK
{"pc_number": 1, "pesos_added": 5, "minutes_added": 30, "total_minutes": 60, "session_token": "tok_abc123"}
Response 400 — No matching rate
{"detail": "No rate configured for ₱5"}
Response 404 — PC not found
{"detail": "PC 1 not found"}

📊 Dashboard Endpoints 🍪 Session Cookie

The dashboard uses a pisonet_session JWT cookie set on login. These endpoints are called by the admin UI JavaScript and are only accessible from a logged-in browser session.

POST /dashboard/login Admin login — sets session cookie ▶

Form submission with username and password. On success: redirects to /dashboard and sets pisonet_session cookie (HTTPOnly, SameSite=lax, 8h). On failure: re-renders the login page with an error.

GET /dashboard/monitor/status Live PC status for monitor page ▶

Called by the monitor page every 3 seconds via JavaScript. Returns all PC statuses including screenshot availability.

Response 200 OK
[
  {"pc_number": 1, "name": "PC-01", "is_online": true, "is_locked": false,
   "remaining_minutes": 28, "remaining_seconds": 45, "remaining_total_sec": 1725, "has_screenshot": true}
]
POST /dashboard/api/pc/add-time Add minutes from dashboard UI ▶
